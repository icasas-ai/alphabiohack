Roles And Access

The current product uses four application roles, plus a company membership layer. In practice, the operating model is clear: patients book, front desk operates appointments, therapists run the practice workflow, and admins have broad internal access.

Application roles

  • Patient: public site, booking, own appointments, own profile
  • FrontDesk: operates appointments for the managed therapist, but not locations, availability, specialties, or personnel
  • Therapist: dashboard, bookings, availability, specialties, locations, personnel, and profile
  • Admin: elevated internal operator with broad access across therapist-managed surfaces

What each role can do

  • Appointments can be operated by FrontDesk, Therapist, and Admin
  • Availability can be managed by Therapist and Admin
  • Personnel can be managed by Therapist and Admin
  • Protected specialty, location, and company flows are intended for operator roles, not patients

Company membership layer

The database also has company membership roles: Owner, Therapist, FrontDesk, and Patient. The most important nuance is that Owner is a company membership role, not a separate application role.
